SSH Brute-Force Detection — Windows OpenSSH + Wazuh

WazuhWindows OpenSSHKali LinuxMITRE T1110

Built a small SOC home lab to detect SSH brute-force attacks against a Windows 10 machine running OpenSSH Server, using Wazuh for log correlation and custom rule authoring.

Wazuh Server→ Windows 10 + OpenSSH (target)→ Kali Linux (attacker)

Objective: detect repeated SSH authentication failures and raise a high-severity alert on a brute-force pattern.

Custom rule — 100100 (SSH failed login): filters Windows events generated specifically by sshd.exe, inheriting from base rule 60122.

<rule id="100100" level="5"> <if_sid>60122</if_sid> <field name="win.eventdata.processName">sshd.exe</field> <description> SSH (Windows OpenSSH): failed password attempt for user $(win.eventdata.targetUserName) </description> <mitre><id>T1110</id></mitre> <group>authentication_failed,sshd,</group> </rule>

Custom rule — 100101 (SSH brute-force): correlates repeated 100100 hits against the same username within a time window.

<rule id="100101" level="10" frequency="6" timeframe="120"> <if_matched_sid>100100</if_matched_sid> <same_field>win.eventdata.targetUserName</same_field> <description> SSH (Windows OpenSSH) brute force: multiple failed password attempts for user $(win.eventdata.targetUserName) </description> <mitre><id>T1110</id></mitre> <group>authentication_failures,sshd,</group> </rule>

Triggers on 6 failed logins, against the same username, within 120 seconds.

Attack simulation: launched repeated failed SSH login attempts from Kali Linux against the Windows target.

  • Rule 100100 fired on each individual failed password attempt.
  • Rule 100101 fired as the high-severity correlated brute-force alert.

MITRE ATT&CK mapping: T1110 — Brute Force, covering password-guessing attacks against remote services such as SSH.

Key takeaways:

  • Understanding Wazuh rule inheritance via if_sid
  • Event correlation using frequency and timeframe
  • Customizing local_rules.xml
  • Detecting Windows OpenSSH attacks instead of relying on generic failed-logon rules
  • Mapping detections to the MITRE ATT&CK framework

SSH Auth Log Analyzer — Brute-Force Flagging Script

PythonPandasLog Analysis

A standalone Python script that parses SSH auth.log files, flags IPs that exceed a failed-login threshold within a sliding time window, and visualizes the noisiest offenders.

  • Regex-based parsing of "Failed password" entries with timestamp and source IP extraction.
  • Sliding-window detection (default: 5+ failures within 5 minutes) per source IP.
  • Exports flagged IPs to alerts.csv and renders a horizontal bar chart of top offending IPs.
#!/usr/bin/env python3
"""
Simple SSH Log Analyzer — flags brute-force IPs.
Edit LOG_FILE below, then just run: python simple_log_ids.py
"""

import re
import pandas as pd
import matplotlib.pyplot as plt

# ---- EDIT THESE ----
LOG_FILE = r"E:\pyfiles\ssh log analyzer\auth.log"      # path to your SSH log
FAILED_THRESHOLD = 5        # alerts if an IP fails this many times...
WINDOW_MINUTES = 5          # ...within this many minutes
# ---------------------

SSH_RE = re.compile(r'(?P<time>\w{3}\s+\d+ \d{2}:\d{2}:\d{2}).*Failed password.*from (?P<ip>\d+\.\d+\.\d+\.\d+)')

def parse_log():
    rows = []
    with open(LOG_FILE, errors="ignore") as f:
        for line in f:
            m = SSH_RE.search(line)
            if m:
                ts = pd.to_datetime(f"2026 {m['time']}", format="%Y %b %d %H:%M:%S")
                rows.append((m["ip"], ts))
    return pd.DataFrame(rows, columns=["ip", "time"])

def find_alerts(df):
    alerts = []
    window = pd.Timedelta(minutes=WINDOW_MINUTES)
    for ip, group in df.groupby("ip"):
        times = group["time"].sort_values().tolist()
        i = 0
        for j in range(len(times)):
            while times[j] - times[i] > window:
                i += 1
            if j - i + 1 >= FAILED_THRESHOLD:
                # count = total failed attempts for this IP (matches chart),
                # window is only used to detect/qualify the burst
                alerts.append({"ip": ip, "count": len(times), "first": times[i], "last": times[j]})
                break
    return pd.DataFrame(alerts)

def main():
    df = parse_log()
    print(f"Parsed {len(df)} matching lines from {LOG_FILE}")

    alerts = find_alerts(df)
    if alerts.empty:
        print("No suspicious activity found.")
    else:
        print(f"\n{len(alerts)} suspicious IP(s):")
        print(alerts.to_string(index=False))
        alerts.to_csv("alerts.csv", index=False)
        print("\nSaved to alerts.csv")

    # quick chart: top IPs by failed-login count
    top = df["ip"].value_counts().head(10)
    top.plot(kind="barh", title="Top IPs by Failed SSH Logins")
    plt.xlabel("Failed attempts")
    plt.gca().invert_yaxis()
    plt.tight_layout()
    plt.savefig("top_ips.png")
    print("Chart saved to top_ips.png")

if __name__ == "__main__":
    main()

Built with the help of AI — used as a learning exercise to understand log parsing, sliding-window detection logic, and basic data visualization with pandas/matplotlib.
